## Correções aplicadas

### R-02 — TC-D6: expectativa de status errada (DashboardIntegrationTest)

- **Arquivo:** `backend/src/test/java/com/gastos/DashboardIntegrationTest.java`
- **Problema:** O teste esperava `400 Bad Request` quando o parâmetro `month`
  não era informado, mas o controller retorna `200 OK` com fallback para o mês atual.
- **Correção:** Alterado `status().isBadRequest()` para `status().isOk()`.
  Atualizado também o nome do método (`returns400` → `returns200`) e o `@DisplayName`.

### R-03 — TC-D8: caminho JSONPath errado (DashboardIntegrationTest)

- **Arquivo:** `backend/src/test/java/com/gastos/DashboardIntegrationTest.java`
- **Problema:** O teste referenciava `$.byCategory` mas o campo correto na
  resposta JSON do DashboardController é `$.expenseByCategory`.
- **Correção:** Substituídas todas as 3 ocorrências de `$.byCategory[...]`
  por `$.expenseByCategory[...]`.

### R-04 — Javadoc com limiar WARNING incorreto (BudgetIntegrationTest)

- **Arquivo:** `backend/src/test/java/com/gastos/BudgetIntegrationTest.java`
- **Problema:** O Javadoc da classe dizia "WARNING se > 75%", mas a regra de
  negócio correta é "WARNING se > 70%".
- **Correção:** Alterado o comentário de `<= 75` / `> 75` para `<= 70` / `> 70`.
