# Checkpoint Arquiteto — 2026-05-29

## Feito nesta sessão

### Contratos de API definidos (docs/api.md atualizado)

**Fatia 4 — Dashboard (`GET /api/dashboard`):**
- Response completo especificado: totalIncome, totalExpense, balance, previousMonth,
  incomeVariation, expenseVariation, expenseByCategory (com %), recentTransactions (últimos 10).
- Regras de agregação documentadas e fechadas:
  - Parcelados (CREDIT + installmentsTotal > 1): usa `installment.amount` WHERE `reference_month` = mês
  - À vista e receitas: usa `transaction.amount` WHERE `transaction.date` no mês
- Tratamento de divisão por zero: `incomeVariation` e `expenseVariation` retornam `null`
  quando mês anterior tem total zero.
- `expenseByCategory.percentage` = total_categoria / totalExpense * 100; lista vazia se totalExpense = 0.
- Status codes: 200, 400 (month ausente ou inválido), 401.

**Fatia 5 — Orçamento (`GET/POST/PUT/DELETE /api/budgets`):**
- `BudgetResponse` especificado com campos: id, categoryId, categoryName, categoryColor,
  month (yyyy-MM-01), limitAmount, spentAmount, remainingAmount, percentage, status, version.
- Status calculado: OK (≤70%), WARNING (>70%–≤100%), EXCEEDED (>100%).
- `spentAmount` usa mesma regra de agregação do dashboard (consistência garantida).
- Optimistic locking via `version` em PUT (retorna 409 se conflito).
- Validações: categoria ativa + tipo EXPENSE/BOTH, unicidade category+month (409), limitAmount > 0.
- Status codes completos para todos os 4 endpoints.
