## O que foi feito

### Migration V4 criada

Arquivo: `backend/src/main/resources/db/migration/V4__add_category_inactive.sql`

Conteúdo:
- `ALTER TABLE category ADD COLUMN inactive BOOLEAN NOT NULL DEFAULT false;`
  Sem necessidade de UPDATE nos registros existentes — o DEFAULT garante que
  todas as 12 categorias seeds (V3) permaneçam ativas automaticamente.
- `CREATE INDEX idx_category_active ON category (id) WHERE inactive = false;`
  Índice parcial para otimizar o `findAllActive` que o backend executará com
  frequência. Menor footprint e mais seletivo que um índice total.

### Verificações realizadas

- Nomenclatura: `V4__add_category_inactive.sql` — segue convenção Flyway `V{n}__descricao_em_ingles.sql`.
- Versão sequencial: V1, V2, V3 já existiam; V4 é o próximo número correto.
- Migrações V1, V2, V3: NÃO foram alteradas.
- SQL válido para PostgreSQL 16: sintaxe `ALTER TABLE ... ADD COLUMN`, índice parcial com `WHERE` e `DEFAULT` em coluna booleana são todos suportados.
- Compatibilidade com dados existentes: `DEFAULT false` aplicado em nível de DDL; zero downtime em banco vazio (docker compose do zero roda limpamente).

### Decisão referenciada

`memory/decisions/2026-05-27-delete-category.md` — soft delete escolhido para
preservar integridade referencial em `transaction`, `budget` e `recurring_rule`
sem alterar FKs existentes.
