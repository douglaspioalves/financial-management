## O que foi feito

### 1. Modelo de dados atualizado
- `frontend/src/app/core/models/transaction.models.ts`
  - Adicionada interface `CardSummary` (substituindo campo `cardId: string` por `card?: CardSummary | null`)
  - Adicionada interface `InstallmentResponse` com campos `id`, `number`, `amount`, `referenceMonth`

### 2. Serviço HTTP criado
- `frontend/src/app/transactions/services/installment.service.ts`
  - Método `getInstallments(transactionId: string): Observable<InstallmentResponse[]>`
  - Endpoint: `GET /api/transactions/{id}/installments`

### 3. Badge "X/N" na lista de lançamentos
- Chip/badge em lilás (#b8a9d9) visível para transações com `installmentsTotal > 1`
- Exibe "parcela_atual/total" (ex.: "1/3")
- Número da parcela atual calculado por correspondência com `referenceMonth` (após carregamento) ou estimado pela diferença de meses (antes do carregamento)
- Tooltip mostrando "Parcela X de N"

### 4. Expansion panel com lista de parcelas
- `MatExpansionModule` adicionado ao `TransactionListComponent`
- Lazy loading: parcelas carregadas apenas ao expandir o painel
- Spinner durante o carregamento
- Tabela com colunas: # | Mês de referência | Valor
- Destaque verde-menta suave (`#7fc4a0` com 20% opacidade) na parcela do mês atual
- Mês formatado como "mmm/yyyy" em pt-br (ex.: "nov/2026")
- Valor formatado em moeda pt-br (R$ 33,33)
- Erro HTTP exibe snackbar "Erro ao carregar parcelas."

### 5. Compatibilidade modo escuro
- Seletores `:host-context(body.dark-theme)` ajustam cores do badge lilás e destaque verde-menta

### 6. Correções
- `transaction-form.component.ts`: `tx.cardId` → `tx.card?.id` (modelo novo)
- `angular.json`: budget de estilo aumentado para 8kB warning / 16kB error
